# 📅 프로젝트 Interface 개발 작업 일지 (2026.02.13 ~ 2026.02.18)

이 문서는 해당 기간 동안 **Project Interface**에서 진행된 주요 수정 사항을 정리한 것입니다. 기술적인 내용보다는 **"어떤 문제를 해결했는지(Why)"**와 **"어떻게 해결했는지(How)"**를 중심으로, 누구나 이해하기 쉽도록 작성되었습니다.

---

## 📅 2월 13일 (금)
### 1. 외부 데이터 연동(MCP) 강화 및 병렬 처리 구조 변경
*   **왜 수정했나요? (Why)**
    *   **과거 방식 (Before)**: 뉴스 기사 수집, 분석, 리포트 작성이 모두 **순차적(1→2→3)**으로 진행되었습니다.
        *   문제점: 앞 단계인 데이터 수집이 지연되면, 뒷 단계인 리포트 작성도 대기해야 해서 전체 실행 시간이 길어졌습니다.
    *   **외부 도구 연결의 어려움**: AI 모델이 뉴스 검색이나 주식 데이터 등 외부 정보를 가져오려면, 매번 각 도구에 맞는 복잡한 연결 코드를 새로 작성해야 했습니다.
*   **어떻게 바꿨나요? (How)**
    *   **🔌 표준 연결 규격, MCP(Model Context Protocol) 도입**:
        *   **MCP란?**: AI가 다양한 외부 도구(뉴스 검색, 데이터베이스 등)를 쉽고 통일된 방식으로 사용할 수 있게 해주는 **"표준 인터페이스"**입니다.
        *   **효과**: 뉴스 검색, 주가 정보 조회 등 다양한 외부 데이터를 일관된 방식으로 AI에게 제공할 수 있게 되어, 데이터 수집 기능의 확장성과 유지보수성이 크게 향상되었습니다.
    *   **⚡️ 병렬 처리(Parallel Processing) 구조 적용**:
        *   **작업 동시 실행**: 서로 의존성이 없는 작업들을 **동시에 실행**하도록 구조를 변경했습니다.
            1.  **용어 사전(Glossary) 작업**: 어려운 금융 용어를 추출하고 설명하는 단계
            2.  **차트(Chart) 생성 작업**: 데이터를 시각화하는 단계
            *   👉 이 두 작업이 동시에 시작되고 진행됩니다.
        *   **결과**: 불필요한 대기 시간이 사라져, 전체 리포트가 완성되는 속도가 빨라졌습니다.

### 2. 콘텐츠 생성 품질 집중 점검
*   **왜 수정했나요? (Why)**
    *   AI가 글을 쓸 때 우리가 원하는 '톤'과 '형식'을 잘 지키는지 확인이 필요했습니다.
*   **어떻게 바꿨나요? (How)**
    *   주요 토픽 5개를 선정하여 AI에게 글(Interface 2 단계)을 쓰게 시킨 뒤, 결과물이 자연스러운지 사람의 눈으로 꼼꼼히 검수했습니다.

## 📅 2월 14일 (토)
### 1. 시스템 충돌 문제 해결 (동시성 제어)
*   **왜 수정했나요? (Why)**
    *   작업 속도를 높이기 위해 "동시에 여러 작업"을 시켰더니, AI가 데이터를 저장할 때 서로 엉키거나 충돌하여 시스템이 멈추는 현상이 발생했습니다. (마치 여러 사람이 하나의 장부에 동시에 글을 쓰려다 펜이 엉킨 것과 같습니다.)
*   **어떻게 바꿨나요? (How)**
    *   데이터가 꼬이지 않도록 **"교통 정리(Concurrency Control)"** 기능을 추가했습니다.
    *   여러 작업이 동시에 끝나더라도 안전하게 데이터를 합치도록 안전장치를 마련했습니다.

## 📅 2월 15일 (일)
### 1. 병렬 처리 안정화 테스트
*   **왜 수정했나요? (Why)**
    *   전날 만든 "교통 정리" 기능이 대량의 작업에서도 잘 작동하는지, 그리고 AI가 너무 많은 단어를 한꺼번에 쏟아내지는 않는지 확인해야 했습니다.
*   **어떻게 바꿨나요? (How)**
    *   시스템을 처음부터 끝까지 가동하며, 중간에 멈추거나 데이터가 누락되지 않는지 확인했습니다.
    *   용어 정리(Glossary)와 차트 생성(Chart) 작업이 동시에 끝나면, 자연스럽게 다음 단계로 넘어가도록 흐름을 매끄럽게 다듬었습니다.

## 📅 2월 16일 (월)
### 1. 실시간 뉴스 기반 리허설
*   **왜 수정했나요? (Why)**
    *   지금까지는 가짜 데이터(테스트용)로 연습했지만, 이제는 **"진짜 오늘 자 뉴스"**를 넣어도 잘 작동하는지 볼 차례였습니다.
*   **어떻게 바꿨나요? (How)**
    *   실시간 뉴스 수집 모드(`live` 모드)를 켜고 전체 시스템을 돌려봤습니다.
    *   최신 뉴스를 읽고 -> 요약하고 -> 리포트를 작성하는 전 과정이 매끄럽게 성공했습니다.

## 📅 2월 17일 (화)
### 1. AI 모델 버전 최신화
*   **왜 수정했나요? (Why)**
    *   우리가 사용하던 AI 모델의 이름이 구버전이라, 시스템이 "주소를 찾을 수 없다(NotFoundError)"는 오류를 냈습니다.
*   **어떻게 바꿨나요? (How)**
    *   모든 설정 파일에서 AI 모델의 이름을 최신 버전(`claude-sonnet-4...`)으로 교체하여 오류를 해결했습니다.

## 📅 2월 18일 (수 - 오늘)
### 1. 리포트 문체 및 디자인, 안전 규칙 대폭 강화 (`3_pages.md`)
*   **왜 수정했나요? (Why)**
    *   AI가 작성한 글이 너무 딱딱하고 교과서 같아서 읽는 재미가 없었습니다.
    *   글이 한 덩어리로 뭉쳐 있어 가독성이 떨어졌습니다.
    *   금융 콘텐츠 특성상 "없는 사실을 지어내거나(Hallucination)", "무책임하게 투자를 권유"하는 것은 치명적이므로 이를 확실히 막아야 했습니다.
*   **어떻게 바꿨나요? (How)**
    *   **🗣 톤앤매너(Tone & Manner) 전면 수정**: 
        *   딱딱한 "~합니다/습니다" 말투를 금지하고, 무조건 **"~해요", "~거든요"** 같은 **친근한 대화체(해요체)**를 쓰도록 강제했습니다.
        *   독자를 "여러분", "우리"라고 부르며 말을 걸듯이(수사적 질문) 글을 쓰게 했습니다.
    *   **✨ 가독성 디자인(Markdown) 적용**: 
        *   중요한 단어는 **굵게(Bold)** 표시하고, 핵심 정의는 **박스(인용구)**에 넣어 눈에 띄게 만들었습니다.
        *   긴 줄글 대신 **리스트(목록)**를 사용하여 내용을 구조화했습니다.
    *   **🛡 안전 및 윤리 규칙(Safety Rules) 강화**: 
        *   **"근거 우선"**: 확인되지 않은 수치나 날짜는 절대 창작하지 못하게 막았습니다.
        *   **"투자 권유 금지"**: '매수', '강력 추천' 같은 단어 사용을 원천 차단했습니다.
        *   수치가 확실하지 않을 때는 '약', '추정' 같은 표현을 쓰도록 안전장치를 마련했습니다.
    *   **📝 구체적인 예시(Example) 제공**: 
        *   AI가 헷갈리지 않게 각 챕터별(배경, 개념 설명, 과거 사례 등)로 "잘 쓴 글의 예시"를 직접 보여주어 따라 하게 만들었습니다.
